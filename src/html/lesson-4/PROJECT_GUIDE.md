# Final Project: Build Your Own Interactive Webpage

## 🎯 Project Overview

In this final project, you will design and build an **interactive portfolio or showcase website** that demonstrates your mastery of HTML and CSS. You'll create a professional, responsive webpage that combines proper HTML structure, stylish CSS design, and interactive elements.

---

## 📋 Project Options

Choose **ONE** of the following projects:

### Option 1: Personal Portfolio Website
Create a professional portfolio showcasing:
- Your introduction/bio section
- Skills section with visual display (cards, bars, or badges)
- Projects/work samples section
- Contact information and call-to-action
- Responsive design for mobile and desktop

### Option 2: Business/Service Showcase
Build a landing page for a business or service featuring:
- Hero section with eye-catching header
- Services/products section
- About the business section
- Customer testimonials
- Contact form or call-to-action button
- Fully responsive layout

### Option 3: Event Promotion Page
Create an engaging page promoting an event with:
- Event header with key details
- Event schedule/timeline
- Speaker or performer profiles
- Venue information
- Ticket information and booking button
- Responsive design

---

## 🎓 Learning Objectives

By completing this project, you will:
✅ Apply semantic HTML elements for better structure  
✅ Create a responsive layout using Flexbox or CSS Grid  
✅ Style webpages with colors, fonts, and visual hierarchy  
✅ Implement interactive CSS effects (hover, transitions, animations)  
✅ Use CSS best practices (organization, reusability)  
✅ Design for mobile-first responsiveness

## 📐 Technical Requirements

### HTML Requirements
- [ ] Use semantic HTML5 elements (`<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`, etc.)
- [ ] Include a navigation menu (fixed or sticky)
- [ ] Minimum 4 distinct sections/areas
- [ ] Include at least one form element (contact form, newsletter signup, etc.)
- [ ] Include at least one image (hero image, logo, product images, etc.)
- [ ] Proper document structure with `<!DOCTYPE>`, `<html>`, `<head>`, `<body>`
- [ ] Meta tags for character encoding and viewport (responsive)

### CSS Requirements
- [ ] External stylesheet (separate `.css` file)
- [ ] Responsive design that works on mobile (320px), tablet (768px), and desktop (1024px+)
- [ ] At least 3 breakpoints using media queries
- [ ] Use of Flexbox or CSS Grid for layout
- [ ] Custom color scheme with at least 4 colors
- [ ] At least 2 custom fonts (from Google Fonts or system fonts)
- [ ] Consistent spacing and padding throughout
- [ ] Professional visual hierarchy (heading sizes, contrast, etc.)

### Interactive Elements (CSS)
- [ ] At least 3 hover effects on buttons or interactive elements
- [ ] At least 1 CSS transition or animation
- [ ] Smooth scrolling or scroll effects
- [ ] Form styling (input focus states, button hover states)

## 🚀 Getting Started

### Step 1: Plan Your Design
1. Sketch out your layout on paper or wireframe tool
2. Decide on your color scheme
3. List the main sections you need
4. Plan your mobile, tablet, and desktop layouts

### Step 2: Create HTML Structure
1. Start with semantic HTML tags
2. Add content for each section
3. Include a navigation menu
4. Add a form or interactive element
5. Include images with proper `alt` attributes

### Step 3: Create CSS Styling
1. Start with a mobile-first approach (base styles for mobile)
2. Add colors, fonts, and spacing
3. Use Flexbox/Grid for layout
4. Add hover effects and transitions
5. Implement media queries for tablets and desktops

### Step 4: Test Responsiveness
1. Test on mobile (320px width)
2. Test on tablet (768px width)
3. Test on desktop (1024px+ width)
4. Test in different browsers
5. Check form functionality

### Step 5: Polish and Refine
1. Review color scheme and contrast
2. Ensure all interactive elements are obvious
3. Add animations or transitions
4. Optimize images
5. Add comments to your code

## 💡 Design Tips

### Color Scheme
- Choose a primary color and 2-3 complementary colors
- Ensure sufficient contrast for readability
- Use consistent colors throughout

### Typography
- Use 2-3 complementary fonts maximum
- Keep font sizes consistent across sections
- Use heading hierarchy (h1 > h2 > h3)

### Spacing
- Use consistent padding and margins
- Add whitespace for visual breathing room
- Align elements using Flexbox or Grid

### Responsiveness
- Design mobile-first, then enhance for larger screens
- Test navigation on mobile (hamburger menu or simplified nav)
- Ensure text is readable on all screen sizes

### Interactivity
- Make buttons and links obvious (clear call-to-action)
- Add hover effects to interactive elements
- Use transitions for smooth animations
- Provide visual feedback for form inputs
